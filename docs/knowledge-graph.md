## 3. 오더구분 판별 로직

### 우선순위: RE-ORDER → MAIN → SPOT

```python
def classify(style, po_no, date_serial, season, sm, style_has_k001):
    is_reorder_no = po_no and po_no[-1] != '1'
    cat = sm.get(style, {}).get('category', '')

    # 규칙 1: acc 카테고리 → 항상 MAIN (RE-ORDER 제외)
    if cat == 'acc':
        if is_reorder_no and style_has_k001[style]:
            return 'RE-ORDER'
        return 'MAIN'

    # 규칙 2: PO NO 끝 ≠ '1'
    if is_reorder_no:
        if style_has_k001[style]:
            return 'RE-ORDER'          # K0001 있음 → RE-ORDER
        else:
            return _by_date(date_serial, season)  # K0001 없음 → 날짜 기준

    # 규칙 3: K0001 → 발주일자 기준
    return _by_date(date_serial, season)

def _by_date(date_serial, season):
    dt = serial_to_date(date_serial)
    if not dt: return 'UNKNOWN'
    m, y = dt.month, dt.year
    if 6 <= m <= 10:  return 'MAIN'
    if m >= 11 or m <= 3: return 'SPOT'
    # 4~5월 엣지케이스: 연도로 구분
    if season == '26SS': return 'MAIN' if y == 2025 else 'SPOT'
    if season == '25SS': return 'MAIN' if y == 2024 else 'SPOT'
```

## 4. 집계 지표 도출 방법

### KPI 지표

| 지표 | 소스 | 계산 방법 |
|------|------|---------|
| 기획스타일수 | 스타일마스터 | 해당 시즌(26S+26N) unique style_id 수 |
| 발주스타일수 | PO+마스터 조인 | 마스터 통과한 unique 스타일코드 수 |
| 발주수량 | PO.PO 총수량 | 마스터 통과 행 합산 |
| 발주금액 | PO.납품가(원화) × PO.PO 총수량 | 마스터 통과 행 합산 |
| 입고수량 | 입고현황.입고수량 | 유효 그룹발주번호 기준 합산 |
| 입고금액 | 입고현황.가공임(확정원가) | 유효 그룹발주번호 기준 합산 |
| 입고율 | - | 입고수량 / 발주수량 × 100 |

### 복종별 집계
- 복종 코드: 스타일마스터의 `category` 사용 (`acc`, `top`, `bottom`, `down`, `outer`)
- 조인: PO.스타일코드 → 마스터.style_id → 마스터.category

### 성별 집계
- 성별 코드: 스타일마스터의 `gender` 사용 (`women`, `men`)
- 조인: PO.스타일코드 → 마스터.style_id → 마스터.gender

### 발주월 집계
- 기준 컬럼: `합의납기일` (납기 기준 월 분류, 시리얼 → 날짜 변환)
- 월 순서: 7월→8월→9월→10월→11월→12월→1월→2월→3월→4월→5월→6월 (시즌 순)

### 협력사별 집계
- 협력사명: PO.협력사
- 수량기준 입고율: (입고수량 합 / 발주수량 합) × 100
